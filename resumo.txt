Após ter instanciado as máquinas virtuais, realizou-se a preparação do ambiente computacional. Dessa forma, foram feitas os seguintes ajustes nas máquinas workers e masters como pré-requisitos para que o cluster Kubernetes entre em funcionamento, sendo eles: o desligamento do swap para evitar erros no cluster; instalação do Curl versão 7.64.0; instalação do Docker versão 19.03.13; instalação do Kubernetes com a instalação do  Kubelet, Kubectl e Kubeadm na versão 1.19.3; e habilitar o Cgroups no sistema operacional.
Para etapa final da criação do cluster, foi feita a instalação do HAProxy, bem como a realização da sua configuração. Posteriormente, iniciou-se  o cluster colocando o master1 como o primeiro membro. Em seguida, foram inseridos os 2 masters restantes e as 3 máquinas workers no cluster. Finalmente, após habilitar o plugin Weave Net nos 3 masters, o cluster Kubernetes estará operante.
O ambiente virtual de ensino aprendizagem escolhido foi o Moodle[ https://www.moodle.org/], um software de código aberto desenvolvido em PHP, amplamente utiliado nas instituições ensino superior (IES). Com funcionalidades preparadas para criação de cursos da Educação a Distância. Além disso, ele possui comunidades ativas para discussões, tem documentação atualizada em português. A versão utilizada do moodle foi a versão 4.0dev em  PHP 7.2.33, com servidor web Apache 2.4.25 e com banco banco de dados  Mysql 5.7.32.
Na implantação do Moodle no cluster Kubernetes, foi necessário a criação de 7 arquivos, em formato Yaml, que é um formato de linguagem de marcação,com uma  estruturação mais amigável para leitura por humanos,  esses arquivos são utilizados pelo kubernetes para automatização da criação dos recursos necessários para o funcionamento do cluster, entre eles: o Deployments, o Endpoints, o PersistentVolume, o PersistentVolumeClaim, o HorizontalPodAutoscaler e o Service. Em seguida, foi preciso aplicá-los no cluster Kubernetes pelo terminal de uma máquina master.
